\begin{abstract}
We present an end-to-end machine learning pipeline for cancer subtype
discovery and classification applied to bulk RNA-seq data from The Cancer
Genome Atlas (TCGA) Pan-Cancer dataset.  Starting from raw HTSeq-FPKM-UQ
expression matrices across six cancer types (BRCA, LUAD, KIRC, PRAD, COAD,
GBM), we implement four methods: (1)~unsupervised clustering via PCA +
K-Means, (2)~one-vs-rest logistic regression with $\ell_2$-regularisation,
(3)~an RBF-kernel SVM, and (4)~a multi-layer perceptron (MLP) in PyTorch ---
the latter three from scratch in NumPy/PyTorch.  All supervised models are
evaluated using 5-fold stratified cross-validation with preprocessing fit
inside each fold to prevent data leakage.  K-Means achieves ARI\,=\,0.794,
NMI\,=\,0.849; logistic regression reaches macro-F1\,=\,0.9927, RBF-SVM
reaches 0.9986, and the MLP reaches 0.9981 (5-fold CV: $0.9983 \pm 0.0010$)
on the held-out test set.  The near-identical performance of RBF-SVM and MLP
confirms that the non-linear benefit over logistic regression is kernel-based
rather than depth-dependent.  Motivated by TA feedback, we extend the pipeline
with feature importance analysis: LR weights projected to gene space reveal
established clinical markers (NKX2-1/TTF-1 for LUAD, PCA3 for PRAD, GATA3
for BRCA), and a per-class permutation importance heatmap shows that different
PCs carry distinct class-specific separation (PC2 for PRAD, PC1 for COAD/KIRC,
PC4 for GBM).  These findings confirm that the models learn biologically
meaningful signals rather than technical artefacts.
\end{abstract}

\section{Introduction}
\label{sec:intro}

Cancer is not a single disease.  Even within the same organ, tumours can differ
dramatically in molecular profile, prognosis, and response to therapy.  Bulk
RNA-sequencing captures a genome-wide snapshot of gene expression in a tumour
sample, providing a high-dimensional view of its molecular state.  The
challenge we address is fourfold:

\begin{enumerate}[leftmargin=2em,itemsep=1pt]
  \item \textbf{Unsupervised discovery.}  Can meaningful cancer subtypes be
        recovered from unlabelled gene expression data using PCA followed by
        K-Means clustering?
  \item \textbf{Linear classification.}  How accurately can a multi-class
        logistic regression model predict cancer type from PCA-reduced features?
  \item \textbf{Non-linear classification.}  Does an RBF-kernel SVM or deep
        neural network offer a measurable gain over the linear baseline, and do
        the two non-linear methods differ from each other?
  \item \textbf{Biological validity.}  Do the models learn tissue-specific gene
        expression signatures, or could the high accuracy reflect technical
        batch structure that survived normalisation?
\end{enumerate}

Question 4 was raised during TA review and drives the feature importance
analysis in Section~\ref{sec:importance}.  Our framing is informed by the TCGA
Pan-Cancer cell-of-origin classification literature~\cite{hoadley2018}, which
provides an externally validated baseline for comparison; by the IntClust
consensus clustering of breast cancer~\cite{curtis2012}, which directly
motivates our unsupervised K-Means method; and by benchmarks of multi-omics
clustering methods on TCGA data~\cite{rappoport2018}.

\section{Data Preprocessing}
\label{sec:preprocessing}

\paragraph{Loading and filtering.}
We first read only the header of the 1.8\,GB TSV to identify columns matching
our six cancer types, then load only those columns.

\paragraph{Log-transformation and NaN handling.}
Raw FPKM-UQ values are $\log_2(x+1)$ transformed to stabilise variance.
Missing values are filled with 0.

\paragraph{Variance filtering.}
Genes with variance $<10^{-6}$ are removed, leaving \textbf{20,297 gene
features}.

\paragraph{Train / test split.}
15\% of samples are held out as a final test set via stratified random
splitting (seed 42):
\begin{itemize}[leftmargin=2em,itemsep=0pt]
  \item Development set: 3,065 samples (85\%)
  \item Test set (held out): 542 samples (15\%)
\end{itemize}

\paragraph{Standardisation and PCA.}
A single \texttt{StandardScaler} and PCA (50 components) are fit on the
development set and applied to the test set.  For cross-validation, both
transformers are re-fit inside each fold's training partition only, preventing
data leakage.

\section{Methodology and Implementation}
\label{sec:methodology}

Methods 1, 2, and 4 are implemented from scratch in NumPy or PyTorch.
Method 3 (RBF-SVM) uses \texttt{sklearn.svm.SVC} as an ablation baseline.
Scikit-learn is used only for evaluation metrics, data splitting, and
preprocessing utilities.

\subsection{Method 1: PCA + K-Means Clustering}

\paragraph{PCA.}
We implement PCA via truncated SVD: given centered data
$\mathbf{X} \in \mathbb{R}^{n \times p}$, we compute
$\mathbf{X} = \mathbf{U}\mathbf{S}\mathbf{V}^\top$ and retain the top $k=50$
right singular vectors.  Explained variance ratios are computed as
$r_i = \lambda_i / \sum_j \lambda_j$ where $\lambda_i = s_i^2 / (n-1)$, with
the denominator summed over \emph{all} components.  The top 50 PCs capture
\textbf{63.7\%} of total variance.

\paragraph{K-Means.}
Lloyd's algorithm with $k{=}6$, $n\_\text{init}{=}10$ random restarts, and
convergence tolerance $10^{-4}$.  Labels are re-assigned using the final
centroids after convergence.

\subsection{Method 2: Logistic Regression (One-vs-Rest)}

We train $C=6$ binary logistic classifiers with $\ell_2$-regularised
cross-entropy loss:

\begin{equation}
  \mathcal{L}(\mathbf{w}, b) =
    -\frac{1}{n}\sum_{i=1}^n \Bigl[\tilde{y}_i \log \hat{y}_i +
      (1-\tilde{y}_i)\log(1-\hat{y}_i)\Bigr]
    + \frac{\lambda}{2n} \|\mathbf{w}\|_2^2
\end{equation}

Training uses gradient descent with $\eta{=}0.1$ for 1,000 iterations.
Hyperparameter search over $\lambda \in \{0.001, 0.01, 0.1\}$ is performed via
5-fold stratified CV, with preprocessing fit inside each fold.

\subsection{Method 3: RBF-SVM (Ablation Baseline)}

We train a support vector machine with RBF kernel
(\texttt{SVC(kernel=`rbf', C=1.0, gamma=`scale')}) on the same 50 PCA
dimensions.  This model is included specifically to test whether the MLP's
non-linear advantage over LR is due to kernel-level non-linearity (which any
kernel method would capture) or network depth.  The same 5-fold CV protocol
with preprocessing inside each fold is used.

\subsection{Method 4: Multi-Layer Perceptron (MLP)}

Feedforward network implemented in PyTorch:
\begin{equation*}
  \underbrace{50}_{\text{input}} \to
  \underbrace{512}_{\text{Linear+ReLU+BN+Dropout}} \to
  \underbrace{256}_{\text{Linear+ReLU+BN+Dropout}} \to
  \underbrace{6}_{\text{output}}
\end{equation*}
Trained with Adam ($\eta{=}0.001$), batch size 64, 50 epochs,
\texttt{CrossEntropyLoss}.  A fixed seed (\texttt{torch.manual\_seed(42)}) and
a 10\% dev-set monitor split (never the test set) ensure reproducibility and
eliminate test-set peeking.

\subsection{Evaluation Metrics}

\textbf{Clustering:} ARI, NMI, Silhouette score.
\textbf{Classification:} Macro-averaged F1, Precision, Recall; per-class
confusion matrix; 5-fold CV mean $\pm$ std F1.

the IntClust consensus clustering literature~\cite{curtis2012}, which similarly
finds that small, geometrically heterogeneous cohorts require more components

The heatmap reveals a striking structure: the high overall importance of PC2

is not exploiting a single dominant axis but is genuinely using multiple
distinct biological signals encoded in different principal components.

\section{Analysis and Discussion}
\label{sec:discussion}

\subsection{Why Are Accuracies So High?}

The near-perfect classification results ($>$99\% macro-F1) are consistent with
the biology and prior literature:

\begin{itemize}[leftmargin=2em,itemsep=2pt]
  \item \textbf{Tissue-of-origin dominates expression.}  Cancer largely
        preserves tissue-specific transcriptional programmes~\cite{weinstein2013},
        so the discriminative signal is vastly larger than within-type
        heterogeneity.  Hoadley et al.~\cite{hoadley2018} report similar
        accuracy ($>$95\%) classifying TCGA samples by cell of origin using
        multi-omic features, supporting our RNA-seq-only results.
  \item \textbf{Batch correction.}  EBPlusPlus normalisation removes
        sequencing artefacts; remaining variance is predominantly biological.
  \item \textbf{Feature importance validation.}  The top discriminative genes
        are established clinical markers, confirming biological learning
        (Section~\ref{sec:importance}).
\end{itemize}

\subsection{Model Comparison: Linear vs Kernel vs Deep}
\label{sec:model_comparison}

The three supervised models form a natural hierarchy by expressivity.
Figure~\ref{fig:decision_boundary} shows the decision regions of LR and MLP
in PC1--PC2 space.

  \caption{Decision boundaries in PC1--PC2 space (all other PCs fixed at 0).
           \textit{Left}: LR linear boundary.
           \textit{Centre}: MLP curved boundary.
           \textit{Right}: regions where LR $\neq$ MLP (red) --- these are
           the boundary zones that the non-linear model resolves differently.}
  \label{fig:decision_boundary}
\end{figure}

LR achieves test F1\,=\,0.9927 but shows a linear decision boundary that
cannot perfectly separate the PRAD/COAD boundary visible in
Figure~\ref{fig:pca_labels}.  The RBF-SVM (test F1\,=\,0.9986) and MLP (test
F1\,=\,0.9981) are within 0.05\,pp of each other, despite the MLP having
far more parameters.  This tight agreement has a clear interpretation: the
non-linear benefit is \emph{kernel-level} rather than depth-dependent.  The
RBF kernel implicitly maps the 50-D PCA space to a high-dimensional feature
space where the remaining class boundaries become linearly separable, achieving
essentially the same effect as the MLP's nonlinear activations.  Rappoport \&
Shamir~\cite{rappoport2018} similarly find in their benchmark of multi-omics
TCGA methods that kernel-based approaches frequently match or exceed deep
models when the input space is already well-structured.

The MLP's advantage over LR (0.54\,pp) and SVM's advantage (0.60\,pp) are
both modest but consistent.  Given that the task is tissue-of-origin
classification --- intrinsically easier than within-cancer subtype
classification --- the headroom for non-linear improvement is limited by the
near-perfect separability visible in the PCA projection.

\subsection{Unsupervised vs Supervised Performance}

The ARI gap between K-Means (0.794) and supervised classifiers ($>0.99$) is
expected: K-Means has no label information and must discover structure from
geometry alone.  As shown in Figure~\ref{fig:purity}, five of the six clusters
map cleanly to a single cancer type; residual confusion originates almost
entirely from GBM's small and geometrically overlapping cluster.

\subsection{Data Leakage Prevention}

All \texttt{StandardScaler} and PCA fits are performed inside each CV fold's
training partition only.  The test set is never seen until final evaluation,
and the final MLP uses a dev-set monitor split (10\% of dev) for learning-curve
monitoring rather than the test set.

\subsection{Limitations}

\begin{itemize}[leftmargin=2em,itemsep=1pt]
  \item \textbf{Task difficulty.}  Classifying across tissue-of-origin is an
        inherently easy task.  Harder problems (within-BRCA PAM50 subtypes,
        survival prediction) would provide a more discriminating benchmark and
        are a natural next step.
  \item \textbf{GBM sample size.}  GBM's lower LR F1 (0.96 vs 1.00 for all
        other classes) traces to its 168-sample cohort; future work could use
        class-weighted loss or targeted augmentation.
  \item \textbf{Interpretability path.}  The current LR gene projection does
        not account for gene--gene correlations.  Pathway enrichment analysis
        (Gene Ontology, KEGG) on the top gene lists would provide a richer
        biological interpretation.
\end{itemize}

\section{Conclusions}
\label{sec:conclusions}

We implemented and evaluated a complete machine learning pipeline for cancer
type discovery and classification from TCGA Pan-Cancer RNA-seq data, and
extended it with biological validation and model comparison:

\begin{enumerate}[leftmargin=2em,itemsep=2pt]
  \item \textbf{PCA + K-Means} (unsupervised) recovers the six cancer types
        with ARI\,=\,0.794, NMI\,=\,0.849, confirming strong tissue-of-origin
        structure independent of supervised signal.
  \item \textbf{Logistic Regression} achieves test macro-F1\,=\,0.9927
        (CV: $0.9968 \pm 0.0027$), confirming near linear separability in
        50-D PCA space.
  \item \textbf{RBF-SVM} achieves test F1\,=\,0.9986 (CV: $0.9951 \pm 0.0023$),
        matching the MLP within 0.05\,pp and confirming that the non-linear
        benefit over LR is kernel-level rather than depth-dependent.
  \item \textbf{MLP} [512, 256] achieves test F1\,=\,0.9981 (CV:
        $0.9983 \pm 0.0010$), the most stable model across folds.  Its 0.54\,pp
        gain over LR is consistent with a mild non-linear advantage at class
        boundaries already captured by the RBF kernel.
  \item \textbf{Feature importance analysis} confirms biological learning: top
        discriminative genes include NKX2-1/TTF-1 (LUAD), PCA3 (PRAD), GATA3
        and Mammaglobin (BRCA), and SLC22A2/OCT2 (KIRC).  The per-class
        permutation heatmap reveals that PC2 separates PRAD, PC1 separates
        COAD/KIRC, and PC4/5 are the GBM-specific axes.
\end{enumerate}

\section*{Implementation Notes}

Running \texttt{python src/main.py} from the project root regenerates all
figures into a timestamped directory
\texttt{results/runs/run\_YYYYMMDD\_HHMMSS/}.  New extended figures are saved
to \texttt{graphs/new\_figures/} within each run directory.

\begin{itemize}[leftmargin=2em,itemsep=0pt]
  \item \texttt{src/pca.py} --- PCA via truncated SVD (NumPy)
  \item \texttt{src/kmeans.py} --- Lloyd's K-Means with multiple restarts (NumPy)
  \item \texttt{src/logistic\_regression.py} --- OvR LR with $\ell_2$ (NumPy)
  \item \texttt{src/mlp.py} --- feedforward MLP (PyTorch)
  \item \texttt{src/evaluate.py} --- metrics, plotting, and feature importance
  \item \texttt{src/main.py} --- end-to-end pipeline; generates all figures
\end{itemize}
